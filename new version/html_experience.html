A short story of my biggest career success

and how it was made

Pre-implementation era

We need a software!

Once upon a time there was a popular University's museum in a big city with no integrated software to manage bookings, sell souvenirs & tickets online or at the ticket office, rent conference rooms, manage employees working hours...

A change needed to be done to make lifes of both Employees an Clients easier. And it was coming relentlessly fast. Cause from the very top of the University's hierarchy came the request to make online ticket purchase possible.

Human resources

And we need people!

No one really knew how to do this and where to start, me included. Yet somehow I became the one who led and coordinated all that huge effort from the very beggining to the very end.

With the support of many friendly people, both co-workers and contractors, with multiple hair-went-greys, stomach twists and nails bitten we finally made it! You can see (and judge :) the results at the website. Let me tell you how it was done.

Step 2 - Find out that there is such thing as RDBMS...

wich stands for 'Relative Database Management System' - a system to create and manage relative databases.

The most popular systems are nowadays MySQL and PostgreSQL. The latter is more complex, so I'd recommend starting with MySQL. And remember - even the longest journey starts with the 1st step.

Check my hard skills lexicon for more info about MySQL.

Step 3 - Turn Your local machine into a server

using dedicated packages, one of the most popular being XAMMP - a package containing Apache http server, MySQL database and PHP interpeter (and more).

It's open source so get it downloaded and installed. Turn Your PC into a server and get familiar with new possibilities in safe environment before You jump into a real server jungle out there.

Check my hard skills lexicon for more info about XAMPP.

Step 4 - Build Your first database

thanks the courtesy of one phpMyAdmin - a program that provides user-friendly GUI to create and modify databases.

Databases are very reasonable and logical. They consist of columns and rows, just like Excell. Each row is a record containing values (e.g. name, surname, age, sex, nationality - if You build a database of Your app users). The first value should always be the uniqe ID, a number which is actually a primary key to recognize each record.

Check my hard skills lexicon for more info about XAMPP.

Step 5 - Build Your webpage and get it hosted

which takes a hell of a time actually. Plus the knowledge of html-css-javascript trio is a must-have which makes this step one of the hardest.

But if You endure and prevail all the difficulties, Your own website will open the portal to the other universe for You. Name Your website, reserve a domain, get the hosting service for it and a ssl certificate to make the Visitors feel safe.

Now You're ready to make another step.
