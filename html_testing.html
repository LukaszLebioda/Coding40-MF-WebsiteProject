Testing

A glass half-full

cause You were born to test reality

User story, ep. 1: Enlightment

Lukasz has recently discovered the diverse world of IT which had always been there but Lukasz was too blind to see it. The more blind he had been, the more intense his experiencing of IT-environment is nowadays.

After having tried various aspects of the matter he finally realized that all his experiences, all his actions and all the decisions he had made led to this precious moment:

to the resolution of becoming a software tester.

User story, ep. 2: Theory

So he started where many university-graduates think they should start from - buying / borrowing / downloading tons of books. Some he read from cover to cover, some he left unfinished and some he abandoned after taking look at the contents.

Yet despite the efforts of the authors he couldn't truly comprehense what was being written without previous software testing experience whatsoever.

He needed some practice.

User story, ep. 3: More theory

There was one book however, the book of books, the bible, the almanach he inevitably encountered. The ISTQB syllabus.

A merely 100-pages volume to encompass the foundations of knowledge every software tester just can't do without. Definitely not a thrilling bedtime reading piece of art to be honest. But Lukasz continued turning the pages and getting familiar with the tech-vocabulary...

to find out that he has already been testing a lot!

User story, ep. 4: Exploratory & Acceptance tests

Whenever he was playing a video-game on his PC, searching for his favourite tv-series on Netflix or buying christmas gifts online he was actually running kind of manual exploratory tests!

And he carried out some acceptance tests as well, when he was checking the functionalities of the sale & booking application that was about to be implemented at his work (more about that implementation here).

The thought he wasn't a total newbie after all was kind of refreshing.

User story, ep. 5: Test cases with TestLink

So finally he was ready to run some real organized pre-planned manual testing. A TestLink software was exactly what he was looking for to manage the test cases he was about to invent.

Although not super-intuitive, not easy to install and not easy to set up, the tool soon made recording of the test cases really straightforward and enjoyable.

User story, ep. 7: The Cypress breakthrough

One beautiful day Lukasz learned that tests could be automated. It was like falling in love... Bang! And You know the reason You're breathing for...

Luckily he had learned some JavaScript before and got quite familiar with Visual Studio Code. After hours and hours of heavy practice he was finally able to run his first simple tests that actually tested something.

Below they are, ordered as they get more advanced.

Mountain weather report

Scenario: I'm about to set for an expedition to conquer Gerlach summit (The Tatras, Slovakia) in winter. I'd better see the weather report. The test below would check if it is possible on one particular website.

Context: There are only basic Cypress commands and assertions used in this test. Plus some aliases for practice. Without any specified requirements to look up to I found it really enjoyable to conduct.

Environment:
Testing framework: Cypress (v. 11.2.0)
OS: Windows 10 Home 64-bit
Browser: Google chrome

Test Results:
- main page loaded:
- page resized to mobile viewport
- redirected to wheather maps subpage
- searchbox working, mountain name found
- wheather forecast table visible

Source code:
                                                                               
describe("when navigate to the mountain weather page on my mobile, 
viewport: 393 x 851", () => {

        let mountainName = "Gerlach";

    it("should visit the page and make sure it loaded", () => {
        cy.pause()
        cy.visit("https://www.mountain-forecast.com/", {timeout:10000});
        cy.title("include", "Mountain Weather");
        cy.url().should("include", "mountain-forecast")
                .and("eq", "https://www.mountain-forecast.com/")
                .and("contain", "mountain-forecast.com");
    })
    it("should make sure it loaded even more", () => {
        cy.get("img[alt='Mountain-Forecast']").should("be.visible");
        cy.get("a.active").should("exist");
    })
    it("should click on 'Weather maps' button", () => {
        cy.get("a").contains("Weather maps").click({force:true});
        cy.get("div#MM_search").should("be.visible");
    })
    it("should type in mountain name and display forecast table", () => {
        cy.get("#location").as("myFirstAlias")
        cy.get("@myFirstAlias").clear();
        cy.get("@myFirstAlias").type(`${mountainName} {Enter}`, {delay:100});
    })
    it("should scroll down to weather forecast table", () => {
        cy.get('.forecast-table__table-wrapper')
            .scrollIntoView()
            .should("be.visible")
    })
})

Online bookstore searchbox

Scenario: I am about to buy a novel of one of my favourite authors: John Grisham. There is an online bookstore I get in touch with quite often when I need a good stuff to read. The source code below would see if the searchbox of that bookstore website is flawless.

Context: I used a hook and variables this time. And some new assertions as well. This test is an automated version of manual test I run using TestLink (above). I found it much more difficult to follow the established route of test cases rather than to design them freely. And again - the test failed because it was about to fail. I'm sure the searchbox actually works as planned and designed.

Environment:
Testing framework: Cypress (v. 11.2.0)
OS: Windows 10 Home 64-bit
Browser: Google chrome

Test results:
- main page loaded, searchbox visible
- no search phrase typed in, 'zero result' info provided
- search phrase remains in the searchbox
- search phrase found, results returned
- search phrase not found, info 'zero results' provided

Source code:
                                                                               
describe('when navigate to the online bookstore page', () => {

    let author = "Grisham";
    let dummyPhrase = "sdgrdthfhjgfhkgkgh";
    let searchbox = "#headerSearchInput";
    let website = "https://bonito.pl/";
  
    beforeEach(() => {
        cy.visit(website);
        cy.url().should("contain", "bonito");
    })
  
    it("website loaded and searchbox with placeholder visible", () => {
        cy.get(searchbox).should("be.visible");
        cy.get(searchbox).should("have.attr", "placeholder", "Czego szukasz?");
    })
    it("search icon clicked: no redirection", () => {
        cy.get('.menu_ikona')
            .should("be.visible")
            .click();
        cy.url().should("eq", website);
    })
    it("search icon clicked: no-results info provided", () => {
        cy.get(".menu_ikona")
            .should("be.visible")
            .click();
        cy.get(searchbox).should("have.attr", "placeholder", "Wypełnij to pole");
    })
    it("author's name typed in (no Enter), pop up window with hints appears", () => {
        cy.get(searchbox).clear();
        cy.get(searchbox).type(author);
        cy.get("iframe").should("be.visible")
        cy.get(searchbox).should("have.attr", "placeholder", author);
    })
    it("author's name typed in, {Enter}, 24 book items exist, pagination visible", () => {
        cy.get(searchbox).clear();
        cy.get(searchbox).type(`${author} {Enter}`);
        cy.get("h1").contains(author);
        cy.get('.product_card_min_white').should("have.length", 24);
        cy.get("div.H4B").should("be.visible");
    })
    it("dummy phrase typed in, {Enter}, no-results info provided", () => {
        cy.get(searchbox).clear();
        cy.get(searchbox).type(`${dummyPhrase} {Enter}`);
        cy.get('.ps-1 > .T2L').should("have.text", "0 wyników");
    })
  })

Login / Logout E2E

Scenario: I would like to log in to a banking application. I probably shouldn't try this with a real app. So I found a fake one where people like me can have a go without worrying to much about the consequences.

Context: This time I did not invent the test scenario on my own. I followed a Udemy tutorial that uses a fake banking application to practice login / logout functionalities. This made me got familiar with the concept of fixtures and allow me to use them in my testing practice for the first time.

Environment:
Testing framework: Cypress (v. 11.2.0)
OS: Windows 10 Home 64-bit
Browser: Google chrome

Test results:
- when logging in with invalid credentials, an error message is displayed
- loggin in with valid credentials possible
- logout possible

Source code:
                                                                                 
    /* FIXTURES Used:
    {
        "username_ID": "username",
        "password_ID": "password"
    }
    */
    
    describe("Login / Logout test", () => {
    
        let invalidUsername = "some username";
        let invalidPassword = "some password";
        const errorMessage = "Login and/or password are wrong.";
        
        before( () => {
            cy.visit("http://zero.webappsecurity.com/index.html")
            // 3 shoulds do wyboru: eq = equal, include, contain
            cy.url().should("include", "index.html")
            cy.url().should("contain", "index.html")
            cy.title().should("equal", "Zero - Personal Banking - Loans - Credit Cards") 
            cy.get("#signin_button").click()
        })
    
    
        it("should try to login with invalid data", () => {
            cy.get("#login_form").should("be.visible")
            cy.get("#user_login").clear().type(invalidUsername, {delay: 100})
            cy.get("#user_password").clear().type(invalidPassword, {delay: 100})
            cy.get("#user_remember_me").check() // optional
            cy.get("input[name='submit']").click()
    
        })
    
        it("should display error message", () => {
            cy.get(".alert-error")
                .should("exist")
                .and("contain", errorMessage)
        })
    
        it("should log in to the application", () => {
    
            // fixtures
            cy.fixture("credentials").then(credentials => {
                const username = credentials.username_ID;
                const password = credentials.password_ID; 
    
                // aliases
                cy.get("#user_login").as("loginInput")
                cy.get("@loginInput").clear().type(username, {delay: 100})
                cy.get("#user_password").as("passwordInput")
                cy.get("@passwordInput").clear().type(password, {delay: 100})
    
                cy.get("#user_remember_me").check() // optional
                cy.get("input[name='submit']").click()
            })
    
            cy.get("ul.nav-tabs").should("be.visible")
            
        })
    
        it("should logout from the application", () => {
            cy.contains("username").click()
            cy.get("#logout_link").click()
            cy.url().should("include", "index.html")
        })
    
    })
